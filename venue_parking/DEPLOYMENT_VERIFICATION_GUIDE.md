## 🚀 部署验证步骤

### 第一步：确认云函数已部署到云端

1. **打开微信开发者工具**
2. **点击工具栏「云开发」按钮**
3. **进入「云函数」页面**
4. **检查以下云函数是否存在并且状态正常：**
   - ✅ user
   - ✅ venue
   - ✅ booking
   - ✅ parking
   - ✅ course
   - ✅ coach
   - ✅ initAllTestData

5. **如果有云函数未部署，请按以下步骤部署：**
   - 在开发者工具左侧找到 `cloudfunctions` 目录
   - 右键点击未部署的云函数文件夹
   - 选择「上传并部署：云端安装依赖」
   - 等待部署成功提示

---

### 第二步：创建数据库集合

1. **打开云开发控制台** → 数据库
2. **点击「+」按钮创建以下集合**（如果已存在可跳过）：
   - `users`
   - `venues`
   - `bookings`
   - `parking_records`
   - `parking_config`
   - `coaches`
   - `courses`
   - `course_schedules`
   - `course_bookings`

3. **设置数据库权限**（重要！）

   **users 集合**:
   ```json
   {
     "read": "auth.openid == doc._openid",
     "write": "auth.openid == doc._openid"
   }
   ```

   **venues / coaches / courses / course_schedules / parking_config 集合**:
   ```json
   {
     "read": true,
     "write": false
   }
   ```

   **bookings / course_bookings / parking_records 集合**:
   ```json
   {
     "read": "auth.openid == doc._openid",
     "write": "auth.openid == doc._openid"
   }
   ```

---

### 第三步：初始化测试数据（关键步骤）⚡

#### 方式一：使用云开发控制台（推荐）

1. **在云开发控制台，进入「云函数」**
2. **找到并点击 `initAllTestData` 云函数**
3. **点击右上角「测试」按钮**
4. **输入以下JSON参数：**

   ```json
   {
     "action": "all",
     "clear": true
   }
   ```

5. **点击「运行测试」**
6. **等待10-20秒，查看返回结果**

## ⚠️ 常见问题

### Q1: 提示「云开发环境未初始化」

**原因**: 环境ID配置错误或云开发未开通

**解决方案**:
1. 检查 `miniprogram/app.js` 中的环境ID是否正确
2. 在微信开发者工具点击「云开发」，确认环境已创建
3. 确认 `project.config.json` 中的 `appid` 正确

---

### Q2: 云函数调用失败

**原因**: 云函数未部署或部署失败

**解决方案**:
1. 在云开发控制台查看云函数列表
2. 重新上传部署未成功的云函数
3. 查看云函数日志，检查是否有错误

---

### Q3: 数据库操作权限错误

**原因**: 数据库权限规则设置不正确

**解决方案**:
1. 检查每个集合的权限设置
2. 参考 `database/README.md` 重新配置权限
3. 确保用户已登录获取 openid

---

### Q4: 测试数据初始化失败

**原因**: 集合未创建或云函数超时

**解决方案**:
1. 确认9个集合都已创建
2. 使用分批初始化方式
3. 查看云函数日志定位错误

---

### Q5: tabBar图标不显示

**原因**: 图片文件缺失

**解决方案**:
1. 确认 `miniprogram/images/` 目录下有以下图标：
   - tab-home.png / tab-home-active.png
   - tab-venue.png / tab-venue-active.png
   - tab-course.png / tab-course-active.png
   - tab-parking.png / tab-parking-active.png
   - tab-user.png / tab-user-active.png
2. 如果缺失，可以暂时使用占位图标

---

### Q6: 场馆或课程列表为空

**原因**: 测试数据未成功初始化

**解决方案**:
1. 在云开发控制台 → 数据库，检查 venues 和 courses 集合是否有数据
2. 重新运行 initAllTestData 云函数
3. 查看云函数返回结果，确认初始化成功
